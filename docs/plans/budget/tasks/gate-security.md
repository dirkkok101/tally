## Recipe

### Acceptance Checks

- Success, validation, compatibility, storage, replay, history, position, interruption, and restart preserve 0700 directories, 0600 files, owner identity, and zero orphan financial artifacts.
- Seeded canaries in amounts, names, requests, results, keys, and failures appear in none of arguments, tracked files, stderr, logs, telemetry, crash output, reports, or unrelated results.
- Malformed JSON, unknown fields, symlinks, permissive modes, wrong owner, unknown files, outside-root paths, over-limits, and incompatible versions fail before mutation or Ledger read as designed.
- All workflows complete with network denied, no prompt/TTY read, and no daemon, watcher, scheduler, service, port, child process, or background worker before or after invocation.
- Only the explicit structured stdout data result contains requested financial payloads; errors and reports are metadata-only.

### Failure Criteria

- Do NOT rely on log redaction while payloads remain in arguments, errors, crashes, telemetry, Git, or temp files.
- Do NOT skip linux-x64 permission, symlink, path traversal, crash, network-denied, or process-isolation cases.
- Do NOT mark green from unit tests alone; execute the published Native-AOT binary.
