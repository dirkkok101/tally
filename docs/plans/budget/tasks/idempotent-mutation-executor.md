## Recipe

### Acceptance Checks

- Missing mutation key fails before BEGIN IMMEDIATE; the store keeps only SHA-256 key digest, operation/version, canonical request hash, outcome references, and result hash.
- The same key plus equivalent normalized logical input rehydrates the exact original revision and lifecycle event snapshots with no live status, category, or period enrichment.
- The same key plus different operation/version/input returns a stable conflict and changes no plan state.
- Interruption before commit leaves the prior state authoritative; interruption after commit replays the single committed outcome.

### Failure Criteria

- Do NOT store raw idempotency keys, request/response JSON, duplicated amounts, category names, or later live result state.
- Do NOT reserve outside the business transaction, return an in-progress ambiguity, or add a generic cross-module abstraction.
