## Recipe

### Acceptance Checks

- Get returns exact immutable entries, checked total, period, currency, status, attribution, timestamps, and activation/supersession provenance for the revision ID.
- Current category display/lifecycle evidence is supplemental; rename/archive never rewrites stored ID, amount, provenance, or payload hash.
- List returns explicit period state and NoBudgetPlan/no-active distinctions, then all summaries by createdAt and revisionId ascending without entries or live category evidence.
- Closed history remains readable; unknown revision returns stable not-found and mismatched identities are never inferred.
- Both queries propagate cancellation and perform no mutation, idempotency reservation, actuals query, or retained report write.

### Failure Criteria

- Do NOT rewrite history with current names, omit Draft or Superseded revisions, collapse absent/empty/all-zero states, or return non-deterministic order.
- Do NOT call Ledger for list summaries or expose raw SQL rows.
