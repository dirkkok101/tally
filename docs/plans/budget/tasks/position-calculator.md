## Recipe

### Acceptance Checks

- Each actual with a stable category is Budgeted or ZeroBudget when an explicit entry exists, otherwise Unbudgeted; null category is Uncategorized; each ordinal is consumed exactly once.
- Explicit zero entries remain ZeroBudget; omitted categories have null planned/remaining/over; Uncategorized has no invented ID/name/planned values.
- Planned equals checked explicit-entry sum and actual equals checked sum of all four bucket kinds; every bucket and total reconciles to the Ledger snapshot total.
- Remaining is max(planned-actual,0), Over is max(actual-planned,0), signed negative actuals remain exact, and every Int64 operation is checked before result publication.
- Category positions sort by stable ID ascending with Uncategorized last; identical inputs produce semantically identical results and provenance.

### Failure Criteria

- Do NOT use decimal/double arithmetic, clamp refunds, omit zero rows, merge Unbudgeted/Uncategorized, invent plan values, or publish partial output on overflow/integrity failure.
- Do NOT read storage, call Ledger, retain snapshots, add pace/forecast/trend/narrative, or mutate lifecycle state.
