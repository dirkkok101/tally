## Recipe

### Acceptance Checks

- Without a selector, load the period's Active revision or return NoActiveBudgetPlanRevision before Ledger; with a selector, require that readable revision belongs to the supplied period and ZAR before Ledger.
- Query current public category and actual evidence for exactly [startInclusive,endExclusive) mapped to inclusive Ledger dates, consume one complete snapshot, then calculate once.
- Return revision status, period, category contract, Ledger contract/snapshot/generation, calculation schema, ordered category positions, Uncategorized last, and exact totals.
- No matching actuals preserves all planned values with exact zero actuals; later Ledger correction yields a new cited snapshot while the revision stays unchanged.
- Any revision mismatch, Ledger incompatibility/unavailability/expiry/integrity, unknown category, cancellation, overflow, or resource failure returns no partial position and persists no actual/cursor/position/report.

### Failure Criteria

- Do NOT call Ledger before revision-period/currency validation, fabricate a zero plan, query private storage, retain derived data, or translate Ledger errors into weaker success.
- Do NOT add analytics, recommendations, alerts, control decisions, or mutation behavior.
