## Recipe

### Acceptance Checks

- The descriptor set contains exactly the twelve C12 operation IDs with deterministic versioned request, result, error, exit, mutability, idempotency, limit, and compatibility metadata.
- Source-generated JSON rejects unknown fields; malformed, incompatible, missing-actor, and missing-idempotency requests fail before a handler effect.
- Complex input is accepted only through explicit stdin or file JSON and sensitive paths never appear in process arguments.
- Descriptor discovery and unknown-operation handling perform no CLASSIFY, private corpus, or Ledger data read.

### Failure Criteria

- Do NOT merge capabilities behind generic invoke, manage, run, save, update, list, or delete discriminators - rejected by DD-CLASSIFY-CLI-OPERATION-CONTRACT.
- Do NOT add HTTP endpoints, FastEndpoints, reflection scanning, a mediator, or an event bus - rejected by DD-CLASSIFY-APPLICATION-ARCHITECTURE.
- Do NOT serialize private payloads to stdout errors, stderr, logs, or crash diagnostics.
- Do NOT use strings for system-owned operation state when a closed enum is defined - per ADR-CORE-0004.

### Expected Outputs

- ClassifyOperationContracts
- ClassifyJsonContext
- ClassifyOperationModule descriptors
- Pure ClassifyContractMapper boundary

### Constraints

- Reuse OperationRegistry, OperationDescriptor, CommandResult, and process envelopes produced by the Ledger core task.

### Notes

- Final Program wiring is owned by TASK-CLASSIFY-GATE-INT-PUBLIC-CONTRACT after every handler exists.
