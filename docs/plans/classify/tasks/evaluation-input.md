## Recipe

### Acceptance Checks

- The loader validates purpose=evaluation, classification_v1, projection version, category lifecycle, store generation, snapshot expiry, and the 10000-row bound before returning input.
- All pages are buffered in ordinal order; missing or duplicate ordinals, cursor expiry, page failure, incompatible descriptors, exact-limit overflow, or cancellation returns a stable failure and no evaluation input.
- A successful result contains only the bounded public projection fields and one immutable snapshot fingerprint; it creates no CLASSIFY state and performs no Ledger mutation.
- Repeated acquisition of an unchanged snapshot returns byte-equivalent ordered input and fingerprints.

### Failure Criteria

- Do NOT open Ledger storage, infer eligibility, or reimplement Ledger lifecycle and allocation rules - per DD-CLASSIFY-LEDGER-PUBLIC-PROJECTION.
- Do NOT stream partial pages into CLASSIFY persistence or classification evaluation.
- Do NOT retain raw descriptions or page payloads after the command boundary.

### Expected Outputs

- ClassificationEvaluationInputLoader
- Bounded immutable evaluation input

### Constraints

- Propagate CancellationToken across every page request and discard buffered input on any failure.

### Notes

- The evaluation command creates its ID only after this loader returns a complete result.
