## Recipe

### Acceptance Checks

- A supported canonical rule over an active category appends an immutable draft version with owner, reason, scope hash, normalization version, and prior version reference.
- Unsupported fields, predicates, duplicate or empty conditions, invalid values, missing categories, and archived categories return stable field or lifecycle errors and create no activatable version.
- Saving a draft neither changes the active rule-set pointer nor invokes any Ledger category mutation.
- Identical idempotent requests return the stored draft result; conflicting reuse preserves the original row.

### Failure Criteria

- Do NOT implement generic upsert or update a prior rule row in place - rejected by DD-CLASSIFY-STATE-STORE.
- Do NOT activate a rule, grant broad authority, or mutate Ledger from rule.save.
- Do NOT query private Ledger storage or accept a category not proven active through the public contract.
- Do NOT place business validation in ClassifyContractMapper or ClassifyRowMapper.

### Expected Outputs

- SaveClassificationRuleCommand
- ClassificationRuleStore
- Immutable draft persistence

### Constraints

- Expected failures return CommandResult values; all async calls propagate CancellationToken.
