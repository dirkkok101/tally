## Recipe

### Acceptance Checks

- The registry exposes only description.normalized, account.id, amount.direction, and amount.absolute_minor with the exact predicates in DM-CLASSIFY-RULE-VOCABULARY.
- NormalizerV1 applies Unicode NFKC, invariant case folding, punctuation boundaries, and whitespace collapse while preserving digits and token order.
- Unknown fields, predicates, values, duplicate conditions, empty rules, over-length values, and invalid minor-unit ranges return stable field-level errors.
- Canonical descriptors and condition hashes are byte-stable across repeated runs and culture settings.

### Failure Criteria

- Do NOT add OR, NOT, priority, regex, fuzzy, wildcard, script, plugin, arbitrary expression, or model-score behavior - rejected by DD-CLASSIFY-RULE-VOCABULARY.
- Do NOT remove dates, amounts, reference numbers, stop words, digits, or token order in normalization v1.
- Do NOT delegate rule interpretation to the AI Agent Host.
- Do NOT broaden the grammar if tests or private evidence fail; route back to design.

### Expected Outputs

- ClassificationRuleVocabulary
- NormalizerV1
- Canonical RuleCondition values

### Constraints

- Use closed enums and exact Int64 minor units; pure domain code has no storage or Ledger dependency.

### Notes

- This task builds only the exact designed vocabulary; viability is decided by the later evidence gate.
