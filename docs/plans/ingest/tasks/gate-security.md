## Recipe

### Acceptance Checks

- Published preview receives sourcePath only through private stdin/file JSON; process command line, environment diagnostics, stdout errors, stderr, logs, crash output, and telemetry contain none of the fixture path, canary, row, description, amount, balance, full identifier, request, manifest, or receipt payload.
- INGEST directories are 0700 and database/WAL/SHM/lock/atomic files are 0600 where supported; permission-establishment failure returns stable permission error before sensitive persistence or Ledger mutation.
- Before/after SHA-256 proves every private source unchanged across preview success, malformed/unsupported/resource failure, approval, commit, resume, abandonment, cleanup, cancellation, and injected crash.
- Malformed, scan-only, password-required, ambiguous, active-content/external-reference, decompression, nesting, byte, page, record, memory, and wall-time cases fail closed before mutation and start no network listener, child process, renderer, OCR, or remote fetch.
- Normal extraction creates no temporary; crash/startup cleanup removes only known unlocked disposable INGEST artifacts and leaves no orphan extraction temporary.
- Architecture guards reject HTTP/FastEndpoints/Aspire/PostgreSQL/EF, reflection/plugin loading, private Ledger storage/handler references, payload logging, and source argument options.

### Failure Criteria

- Do NOT weaken canary assertions, redact after logging, or treat process isolation as sufficient — prevent payload entry at each boundary.
- Do NOT skip owner-only mode checks on supported hosts or continue after failure.
- Do NOT persist test fixture paths/content or emit them when a security assertion fails.

### Expected Outputs

- IngestSecurityGateEvidence
- Published security boundary tests
- Malformed/resource canary tests

### Constraints

- Security failures themselves must use metadata-only stable identifiers and never echo the triggering payload.
