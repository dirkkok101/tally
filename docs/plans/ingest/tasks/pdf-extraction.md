## Recipe

### Acceptance Checks

- src/Tally/Tally.csproj pins package PdfPig at exactly 0.1.14 and introduces no legacy UglyToad.PdfPig package, Poppler executable, custom PDF parser, OCR library, or dynamic-code dependency.
- PdfStatementTextExtractor accepts a bounded immutable byte buffer, CancellationToken, and extraction limits and returns ordered PdfDocumentEvidence page/glyph/text/coordinate data with no PdfPig type escaping Infrastructure/Ingest/Pdf.
- The extractor never follows links, renders, executes scripts or embedded objects, performs remote fetch, writes extracted text to disk, or modifies the caller-owned source.
- Unsupported opening-password encryption, scan-only content, malformed structure, ambiguous content order, and every configured byte/page/glyph/time bound return a stable pre-mutation IngestError.
- PrivateStatementFixtureSet reads only the path named by TALLY_INGEST_PRIVATE_FIXTURE_MANIFEST; the ignored manifest supplies three source paths, variant IDs, and expected private facts, and neither paths nor facts enter source control or test output.
- All three private fixtures, including the permission-encrypted fixture, extract non-empty evidence twice with identical digests; Release linux-x64 Native-AOT publish succeeds with zero trim/dynamic-code warnings.

### Failure Criteria

- Do NOT use legacy UglyToad.PdfPig 1.7.0-custom-5, Poppler, a child process, or a custom parser — rejected per DD-INGEST-DOCUMENT-EXTRACTION.
- Do NOT introduce a generic PDF parser interface; PdfStatementTextExtractor remains concrete until a second real production implementation exists.
- Do NOT place source paths in process arguments, logs, test names, snapshots, or failure messages — per DD-INGEST-ARTIFACT-SECURITY.
- Do NOT commit fixture bytes, fixture manifests, filenames, expected private facts, or extracted text.

### Expected Outputs

- Pinned PdfPig dependency
- PdfStatementTextExtractor
- PdfDocumentEvidence and IStatementAdapter seam
- Private fixture injection helper
- Native-AOT extraction tests

### Constraints

- Hard limits are checked before allocation or continued parsing where the source makes that possible; cancellation is propagated at every async boundary.

### Notes

- The ignored private fixture manifest is external test evidence, not a tracked deliverable.
