## Recipe

### Acceptance Checks

- A clean client discovers exactly eight operations, request/result schemas, two variant descriptors, limits, versions, stable errors/exits, examples, and next-operation semantics without source/store access.
- The client constructs valid stdin/file JSON, invokes preview through cleanup non-interactively, receives exactly one stdout envelope per call, uses metadata-only stderr, and branches on structured outcome/exit rather than prose.
- Unknown operation/schema returns help plus stable usage failure; malformed/unknown-field input fails before source processing.
- Incompatible client/INGEST/adapter/manifest/Ledger versions fail before parsing or mutation.
- Expected domain failures return module-scoped stable error and non-zero documented exit without stack trace.
- After output loss the client cannot construct a new identity to continue; status/resume with the original batch recovers safely and no daemon, prompt, network, or source reread is required.

### Failure Criteria

- Do NOT use internal types, database inspection, interactive prompts, prose scraping, or reconstructed candidate IDs in the agent harness.
- Do NOT add bundled skill content not generated from the same descriptors.
