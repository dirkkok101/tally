## Recipe

### Acceptance Checks

- The caller freezes TimeProvider.GetUtcNow once before producer reads and AnalysisWindowPolicy converts it with fixed IANA Africa/Johannesburg independent of host locale.
- Day is the As-of local date intersected with the Budget Period; week is Monday through Sunday clipped to the period; month-to-date is period start through the earlier of local As-of date or period end, inclusive.
- A completed historical period returns the full period, a future period returns NotStarted with no zero-spend interpretation, and rolling-day or rolling-week requests fail before financial processing.
- Window membership uses LEDGER Effective Date and historical generation states that current corrected evidence was used rather than claiming source-state reconstruction.

### Failure Criteria

- Do NOT accept a caller-supplied historical As-of Instant â€” rejected per DD-INSIGHTS-TRUSTED-WINDOW-PACE-POLICY.
- Do NOT use host locale, posting time, invocation time per member, rolling windows, or inferred daily/weekly sub-budgets.
- Do NOT read TimeProvider more than once for one report or hide NotStarted as an empty successful window.

### Expected Outputs

- AnalysisWindowPolicy
- Calendar conformance tests

### Constraints

- Use DateOnly boundaries and explicit inclusive semantics.
