## Recipe

### Acceptance Checks

- BudgetContractClient resolves the released budget.insights.evidence.get descriptor from OperationRegistry and invokes it once through the shared public operation executor with CancellationToken.
- Supported BoundRevision, NoBudgetPlan, and NoActiveBudgetPlanRevision results deserialize through source-generated contracts without changing their plan-state, member, total, snapshot, generation, version, or binding evidence.
- Unsupported BUDGET or LEDGER versions, Source State Changed, resource limits, malformed results, and executor failures return stable typed INSIGHTS failures before report output or retained writes.
- The client exposes no mutation descriptor and performs no retry that could join evidence from separate producer captures.

### Failure Criteria

- Do NOT add an IBudgetContractClient interface or transport adapter — the concrete client over the existing shared executor is selected per DD-INSIGHTS-APPLICATION-ARCHITECTURE.
- Do NOT call budget.position.get plus ledger.actuals.query, resume a cursor, or calculate Budget Position — rejected per DD-INSIGHTS-COHERENT-PUBLIC-EVIDENCE.
- Do NOT open BUDGET or LEDGER storage, import private feature handlers or domain types, grant mutation authority, or log request/result payloads.
