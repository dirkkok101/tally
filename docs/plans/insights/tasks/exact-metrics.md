## Recipe

### Acceptance Checks

- Each requested started day, week, and month-to-date window returns one exact total and, when requested, categories in stable identity order.
- Budgeted, ZeroBudget, Unbudgeted, and Uncategorized are distinct; NoBudgetPlan preserves exact actuals with NoBudgetPlan on plan-relative fields.
- Signed refunds and reversals remain negative, an empty matching set returns exact zero, and total actual equals the checked exact sum of all buckets.
- Every member contributes exactly once per applicable metric; duplicate accounting, overflow, undefined exact operation, or reconciliation mismatch aborts the whole report.

### Failure Criteria

- Do NOT use floating point, clamp refunds to zero, suppress zero results, merge Uncategorized with Unbudgeted, or invent a plan.
- Do NOT expose per-metric public services or move orchestration to callers — rejected per DD-INSIGHTS-DETERMINISTIC-REPORT-COMPILER.
- Do NOT emit partial metric collections after arithmetic or reconciliation failure.

### Expected Outputs

- ExactMetricCalculator
- Exact arithmetic property matrix

### Constraints

- Use checked Int64 minor units and deterministic catalogue/category order.
