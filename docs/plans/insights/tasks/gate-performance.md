## Recipe

### Acceptance Checks

- The fixture contains exactly 100000 active LEDGER transactions through the public BUDGET composite, 1000 category positions, 1000 retained ReportSnapshots, and every active v1 policy without private owner payloads in the report.
- After warm-up, at least 100 measured invocations per operation record p50, p95, maximum duration, peak resident memory, exact result/fingerprint reconciliation, output size, and environment fingerprint.
- Generate and Retain each complete within 5 seconds p95 and 256 MiB peak resident memory; Restate completes within 6 seconds p95; Get, Status, Delete, and a 1000-item metadata List each complete within 1 second p95.
- One-over-limit horizon, member, category, metric, signal, list, or output request fails before financial output, producer over-read, partial report, or retained change.
- Network remains denied, no backup/migration overlaps, List and Status avoid payload reads, exact arithmetic and fingerprints reconcile, and every benchmark class has nonzero discovery.

### Failure Criteria

- Do NOT reduce dataset sizes, measured samples, active policies, category count, or retained history to manufacture a pass.
- Do NOT cache mutable producer truth, weaken exact arithmetic, skip result checks, silently truncate, deserialize payloads for metadata reads, or run a daemon/background worker.
- Do NOT include financial fixture content or private paths in benchmark output.

### Expected Outputs

- InsightsPerformanceGateEvidence
- Payload-free benchmark report

### Constraints

- Run on the supported release host with Release build and record the exact environment fingerprint.
