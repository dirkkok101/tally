## Recipe

### Acceptance Checks

- Seeded financial values, merchant strings, report fragments, private paths, credentials, and disclosure canaries appear only in an explicitly requested successful business result and are absent from arguments, environment echoes, stderr, ordinary logs, telemetry, crash output, Git, shared temporary locations, verification docs, and unrelated operation results.
- INSIGHTS directories are 0700 and database, WAL, SHM, CURRENT, lock, candidate, manifest, backup, restore, receipt, and private evaluation files are 0600 where supported; unsafe owner or mode fails before financial processing or activation.
- Report storage contains bounded derived payloads and stable provenance only; producer members, canonical transactions, raw merchant descriptions, private evaluation cases, PolicyValidationReceipt, and upstream replicas are absent.
- Every accepted workflow succeeds with network denied and no daemon, watcher, scheduler, service, listener, queue, cloud model, child tally process, or background Tally process before, during, or after invocation.
- Static and runtime guards prove INSIGHTS references only public BUDGET/LEDGER contracts and shared operation abstractions and grants no upstream or policy mutation authority.
- Host encrypted-volume support is evidenced as a deployment prerequisite; the report makes no custom encryption claim.

### Failure Criteria

- Do NOT weaken canaries, redact after a leak, serialize DTOs into logs, echo private environment/path values, use permissive fallback modes, or claim encryption not supplied by the host.
- Do NOT allow private database, SQL, feature-handler, repository, domain, HTTP, network, child-process, or background-service dependencies.
- Do NOT accept aggregate security totals unless every public operation, private harness, retained mutation, backup, restore, corrupt-input, and crash partition has nonzero discovery.
