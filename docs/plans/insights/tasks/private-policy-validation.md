## Recipe

### Acceptance Checks

- InsightPolicyValidationHarness accepts the private manifest only from TALLY_INSIGHTS_PRIVATE_EVALUATION_MANIFEST, requires an owner-only regular file, and reads cases in memory without printing paths or payloads.
- Validation records candidate policy, report schema, producer contract, window, corpus, expected-state, and tool fingerprints and accounts for every structural case as supported, unavailable, insufficient, correct/incorrect signal, correct/false severity, or disclosure-canary outcome.
- Sparse, zero, recurring, front-loaded, irregular, refund-heavy, plan-change, category-lifecycle, classification-gap, correction, false-severity, and disclosure-canary shapes are required before success.
- The external PolicyValidationReceipt contains only approved structural IDs, fingerprints, aggregate outcomes, false-high-severity count, canary status, tool version, and owner acceptance; it is never inserted into insights.db.
- Missing evidence blocks changed policy release but does not disable unchanged released policies; deferred policies remain non-executable.
