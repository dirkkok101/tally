## Recipe

### Acceptance Checks

- InsightReportCompiler accepts only normalized request values, one frozen As-of Instant, one BoundInsightEvidence, and released policy descriptors; it performs no producer or storage call.
- The complete report contains budget period, trusted time, exact windows, plan state, exact metrics, every requested signal state, deterministic explanations, limitations, and complete source and policy provenance in released order.
- Repeated compilation with identical inputs produces semantically identical ordered content, contentFingerprint, and reportId; collection order never depends on localized display text.
- InsightReportCanonicalizer serializes the semantic payload without reportId and contentFingerprint, hashes canonical UTF-8 JSON with SHA-256, assigns that hash as contentFingerprint, then derives reportId.
- Any coherence, compatibility, integrity, completeness, exact-arithmetic, canonicalization, or output-limit failure returns no report and no partial financial payload.

### Failure Criteria

- Do NOT embed all calculation inside Generate or expose per-metric orchestration to callers — rejected per DD-INSIGHTS-DETERMINISTIC-REPORT-COMPILER.
- Do NOT read a producer, store state, use reflection serialization, hash derived identity fields into themselves, depend on display text, or emit partial reports.
- Do NOT add Forecast, Trend, Anomaly, or Attention Item formulas while their FRs remain deferred.

### Expected Outputs

- InsightReportCompiler
- InsightReportCanonicalizer
- Determinism and failure-atomicity tests

### Constraints

- The compiler and all collaborators remain concrete pure classes; no IReportCompiler or policy interfaces are introduced.
