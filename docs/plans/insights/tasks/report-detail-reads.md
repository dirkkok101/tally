## Recipe

### Acceptance Checks

- Get for a known identity verifies report schema, payload fingerprint, and content fingerprint and returns the exact immutable ReportSnapshot plus complete stored provenance.
- Explain for a known identity returns only the selected exact stored explanation and provenance and never calls BUDGET, LEDGER, TimeProvider, or InsightReportCompiler.
- Unknown identity returns NotFound without other report metadata; deleted identity returns metadata-only Deleted evidence with no former payload or explanation.
- Every success and expected failure uses a short read connection and changes no snapshot, deletion event, idempotency outcome, recovery receipt, or status state.

### Failure Criteria

- Do NOT recompute current evidence, return list metadata for unknown IDs, expand raw transactions, or deserialize unrelated report payloads.
- Do NOT add a generic repository, HTTP detail endpoint, mutation, cache, or hidden repair on integrity failure â€” rejected per DD-INSIGHTS-APPLICATION-ARCHITECTURE and DD-INSIGHTS-STATE-STORE.
- Do NOT return deleted content from a backup or replay reference.
