## Recipe

### Acceptance Checks

- List accepts only documented BudgetPeriod, trusted creation-range, actor, and count filters; one-over-limit input returns ResourceLimit rather than truncating silently.
- List returns snapshot identity, predecessor identity, BudgetPeriod, trusted creation time, actor, policy versions, content fingerprint, and deletion state ordered by creation time descending then stable identity descending, with no report payload or amount.
- Status returns active generation fingerprint, storage version, integrity state, retained count, deleted count, supported policy versions, and latest durable successful Backup or Restore verification without payloads, private paths, or merchant text.
- Standalone Verify results do not appear in Status and List or Status success and failure change no retained or replay state.
