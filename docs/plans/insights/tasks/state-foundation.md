## Recipe

### Acceptance Checks

- One owner-only CURRENT pointer selects one insights.db inside an owner-only generation directory; directories use 0700 and database, WAL, SHM, pointer, lock, manifest, backup, and recognized candidate files use 0600 where supported.
- Immutable ordered migrations create store_metadata, report_snapshot, report_payload, restatement_link, deletion_event, insights_idempotency_record, recovery_receipt, and restore_event with foreign keys, uniqueness, RESTRICT semantics, and PRAGMA user_version.
- Connections enable foreign_keys, WAL, synchronous FULL, and bounded busy_timeout; writers use BEGIN IMMEDIATE and cancellation-safe rollback; queries use short read connections.
- InsightsRowMapper is a concrete pure mapper between rows and immutable contracts, List and Status metadata paths do not deserialize payloads, and Get verifies schema plus content fingerprints.
- The database never stores dated actual members, raw merchant text, canonical transactions, producer cursors, private corpus rows, arbitrary formulas, PolicyValidationReceipt, report-list payloads, or upstream replicas; tests use real temporary SQLite and never mock SQL.

### Failure Criteria

- Do NOT use JSON files, fully normalize analytical report bodies, add EF Core or a hosted database, or place state in BUDGET/LEDGER — rejected per DD-INSIGHTS-STATE-STORE.
- Do NOT add IInsightsStore, generic repository, provider-neutral unit of work, reflection mapping, multiple writers, or automatic cleanup.
- Do NOT claim custom encryption; host-managed encrypted-volume evidence remains a release gate.
- Do NOT implement this task unless TASK-INSIGHTS-GATE-EVIDENCE-OWNER-VALUE completed successfully.
