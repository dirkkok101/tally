## Recipe

### Acceptance Checks

- Retain validates bounded analysis input, actor, reason, idempotency, permissions, compatibility, and policies, generates internally, and atomically stores one immutable snapshot without accepting caller-authored report content.
- Exact retry returns the same logical snapshot before producer access, changed normalized input conflicts, and validation/generation/storage/cancellation/crash failure leaves no partial snapshot or successful key.
- Get returns the exact immutable snapshot and provenance; bounded List returns deterministic metadata-only history and never includes payload except through explicit Get.
- Ordinary Generate, Get, List, and their failures create no durable state, and later corrections or policy changes cannot modify the retained snapshot.

### Failure Criteria

- Do NOT auto-retain Generate, accept uploaded report content, duplicate a retry, silently truncate List, deserialize payloads for metadata, or mutate stored bytes after source change.
- Do NOT skip fault/restart inspection or infer atomicity from a successful command alone.
