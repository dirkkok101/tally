## Recipe

### Acceptance Checks

- Help and system.schema discovery require no financial input and return exactly eleven operation descriptors plus released policy metadata with stable schemas, versions, authority, mutability, idempotency, limits, errors, exits, and compatibility.
- For each operation, valid stdin or owner-only file input executes exactly one handler, emits exactly one structured business envelope on stdout, and exits with the documented category without prompts or prose parsing.
- Malformed, unknown-field, unsupported version, missing authority/idempotency, NotFound, Deleted, conflict, unavailable, integrity, resource, downstream, and internal paths each emit one stable envelope and no partial financial output or retained change.
- Arguments, stderr, logs, crash output, Git, and shared temporary files remain payload-free; network is denied and no daemon, watcher, scheduler, listener, queue, child tally process, or background process exists.

### Failure Criteria

- Do NOT add interactive fallback, undocumented alias, prose output, multiple business envelopes, financial arguments, hidden authority, HTTP, or background execution.
- Do NOT accept aggregate process totals unless all eleven operations and every exit partition have nonzero discovery.
