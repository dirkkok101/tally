## Recipe

### Acceptance Checks

- Assigning an active category to an active uncategorized transaction appends exactly one active allocation and leaves source facts unchanged.
- Correct with active replacement and reason atomically retires prior allocation and appends one attributable active allocation; history retains both.
- Inactive transaction, missing/archived category, no active allocation, or same-as-active correction returns its stable error with no allocation change.
- Category rename resolves current display name through stable ID for active and retired allocations.
- Identical mutation replay returns the original allocation outcome without a second row.
