## Recipe

### Acceptance Checks

- link-supporting validates active transaction, evidence identity, compatibility, reason, and idempotency before appending one attributable supporting link.
- Identical link replay returns the original result; inactive/missing transaction, incompatible evidence, active link conflict, or changed replay returns a stable error and preserves prior state.
- Supporting linkage never creates confirming statement state, changes amounts/dimensions, or moves a link across supersession.

### Failure Criteria

- Do NOT use a supporting link as a reconciliation decision or allow one evidence item to be silently moved â€” rejected per DD-LEDGER-RECONCILIATION-CONTRACT.
- Do NOT overwrite evidence/link history â€” rejected per DD-LEDGER-IMMUTABLE-HISTORY.
- Do NOT access provider payloads or private INGEST state.
