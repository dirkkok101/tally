## Recipe

### Acceptance Checks

- A redacted matrix covers company-paid discretionary spend, personal after-tax spend, unassigned spend, corrections, refunds, reversals, transfers, voids, supersessions, and at least one candidate mixed purchase.
- For every supported case the report proves one active pool or explicit unassigned state and exact pool/category conservation without deriving pool from account, instrument, cardholder, category, or description.
- OQ-LEDGER-15 resolves only if no representative transaction requires simultaneous split pools; otherwise leave it open, keep this task incomplete, and return to lex:design before pool implementation.

### Failure Criteria

- Do NOT encode pools as categories, infer them from another dimension, or silently split a transaction — rejected per DD-LEDGER-DIMENSIONAL-ATTRIBUTION.
- Do NOT commit raw transaction rows, descriptions, full identifiers, or owner-sensitive mapping data.
- Do NOT resolve the gate by excluding mixed purchases; a demonstrated split-pool need invalidates the current design.

### Expected Outputs

- docs/validation/ledger/oq-15-pool-cardinality.md with redacted scenario matrix
- OQ-LEDGER-15 resolved with a non-empty cardinality decision on the only completing path
