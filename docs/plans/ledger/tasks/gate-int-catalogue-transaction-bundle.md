## Recipe

### Acceptance Checks

- The bundle contains exactly 17 unique descriptors: 11 account/category operations and 6 transaction/allocation/correction operations defined by the consumed modules.
- Descriptor order is operationId ordinal; every CLI path, operationId, kind, contract range, schema, error, and exit entry is retained unchanged from its source module.
- A missing module, duplicate operationId, duplicate CLI path, or incompatible contract range returns a deterministic composition failure before any handler can dispatch.
- Focused tests compare the complete bundle inventory with all four source modules and prove no descriptor is added, dropped, or rewritten.

### Failure Criteria

- Do NOT hand-maintain a second operation schema, discover modules through reflection, or make this gate implement domain behavior.
- Do NOT accept a count-only assertion without exact set equality.
