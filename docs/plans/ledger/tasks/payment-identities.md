## Recipe

### Acceptance Checks

- Create returns stable local ULIDs for instruments and cardholders, accepts owner labels and optional masked suffix/account association only, and rejects full identifiers or provider payloads before mutation.
- Get/list/rename/archive/reactivate return attributable lifecycle history; archived identities retain historical references and reject new attribution until reactivated.
- Duplicate active identity and conflicting idempotency requests return stable conflicts while identical replay returns the original result.

### Failure Criteria

- Do NOT use provider IDs as primary identity, retain full card/account numbers, or infer cardholder from wording — rejected per DD-LEDGER-DIMENSIONAL-ATTRIBUTION.
- Do NOT physically delete referenced identities or overwrite labels/history — rejected per DD-LEDGER-IMMUTABLE-HISTORY.
- Do NOT add a provider adapter or transport vocabulary to contracts.

### Expected Outputs

- Payment identity contracts, policies, stores, handlers, operation modules, and focused tests
