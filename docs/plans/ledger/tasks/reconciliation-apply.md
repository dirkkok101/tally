### Acceptance Checks

- match_existing requires typed owner authority, actor, reason, statement Evidence Record fingerprint, expected advisory projection token, and one target Canonical Transaction; it revalidates current fact, lifecycle, candidate or guard membership, conflicts, and evidence exclusivity inside one BEGIN IMMEDIATE transaction before appending the Reconciliation Decision and confirming link.

- record_ambiguous preserves the complete reviewed candidate set and applies no confirming link or financial effect; record_exception preserves one Reconciliation Exception with stable code and reason; stale, conflicting, changed replay, or unsupported automatic authority preserves prior state.
- Crash injection at every base write boundary leaves the full outcome absent or committed; identical logical statement Evidence Record and disposition replay returns the existing outcome.
